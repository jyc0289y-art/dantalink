## 1. 문제 진술

### 현재 동작 (Phase E 종료 시점)
```
입력 트리거: "샘 알트먼 OpenAI CEO 방한 AI 정상회담 참석"
   ↓
regex r"[가-힣]{1,5}" 토큰화
   ↓
토큰: ["샘"(1자), "알트먼"(3자), "방한", "정상회담", "참석", "한국", "기업과", "협력", "모색"]

   ↓
매칭: 샘표(007540) — score 0.30 × 0.7 페널티 = 0.21
   ↓
matched_on = "샘",  reason = "트리거 토큰 '샘'의 첫 음절 '샘'이 종목명 '샘표'와 일치"
```

```
   ↓

```

| 항목 | 현재 | 의도 |
|------|------|------|
| 토큰 단위 | 1자 한국어 음절 ("샘") | 외국 인명 패턴 ("샘 알트먼") |
| 점수 | 0.21 (1자 페널티) | 0.55 (인명 패턴 = 강한 음운 트리거) |
| 정렬 순위 | Top 30 밖 | Top 5 이내 예상 |
| matched_on 기록 | "샘" | "샘 알트먼" |
| 사용자 가독성 | 왜 샘표인지 불명확 | 인명→종목 매칭 직관 |

## 2. 의도된 매칭의 학술적 근거 (방법론 §2.3)

- **음운적 점화 (Phonological Priming)**: 외국 인명이 미디어에 노출되면 한국인 투자자가 그 인명의 음운적 첫 부분을 자동 활성화 → 유사 음절 종목명과 연관
- **이름 유사성 효과 (Rashes 2001)**: 티커·사명 첫 음절 일치는 단기 주가에 통계적 영향
- **인지적 유창성 (Alter & Oppenheimer 2006)**: 발음하기 쉬운 종목이 더 자주 매수됨

→ "샘 알트먼" 같은 외국 인명이 단타 트리거인 경우, 첫 음절 "샘"은 단순한 1자가 아니라 **인명 전체의 음운 시그니처**임. 1자 토큰 페널티는 적절하지 않음.

### 3.2 알고리즘 변경

```python
import re

# 외국 인명 패턴 — 1-2자 한글(성/이름 첫 글자) + 공백 + 2-4자 한글(이름 본체)
# 예: "샘 알트먼", "팀 쿡", "일론 머스크", "도널드 트럼프"
FOREIGN_NAME_PATTERN = re.compile(r"[가-힣]{1,2}\s+[가-힣]{2,4}")

# 외국 인명은 강한 음운 트리거 → 1자 페널티 면제
_SCORE_FOREIGN_NAME_FIRST1 = 0.40   # 일반 first1(0.30)보다 강함
_SCORE_FOREIGN_NAME_FIRST2 = 0.55   # 정확 일치급
_SCORE_FOREIGN_NAME_EXACT = 0.55


class PhonologicalMatcher(BaseMatcher):
    def match(self, event):
        text = event.title or ""
        candidates = []
        used_syllables = set()  # 같은 첫 음절 중복 차단

        # 1) 외국 인명 패턴 우선 추출 (1자 토큰 페널티 없음)
        for name_match in FOREIGN_NAME_PATTERN.finditer(text):
            full_name = name_match.group()  # 예: "샘 알트먼"
            first_token = full_name.split()[0]  # 예: "샘"
            first_syl = first_token[0]
            if first_syl in used_syllables:
                continue
            used_syllables.add(first_syl)

            stocks = self.db.find_by_first_syllable(first_syl)
            scored = self._score_stocks(first_token, stocks, full_name, is_foreign_name=True)
            for s, score, kind in scored[:self.max_per_token]:
                candidates.append(MatchCandidate(
                    ...,
                    matched_on=full_name,  # "샘 알트먼" 그대로 기록
                    reason=f"외국 인명 '{full_name}'의 첫 음절 '{first_syl}'이 종목명 '{s.name}'과 음운 일치",
                    details={"pattern_kind": "foreign_name", "first_syllable": first_syl, ...},
                ))

        # 2) 일반 토큰 추출 (인명 패턴에 포함되지 않은 단어들)
        # 인명 패턴 영역을 마스킹 후 regex `[가-힣]{2,5}` 적용
        masked = FOREIGN_NAME_PATTERN.sub(lambda m: " " * len(m.group()), text)
        tokens = re.findall(r"[가-힣]{2,5}", masked)  # 1자는 제외 (인명만 1자 시작 허용)
        for tok in tokens:
            if tok in _STOPWORDS_MULTI:
                continue
            if tok[0] in used_syllables:
                continue
            used_syllables.add(tok[0])
            # ... 기존 로직 (페널티 없음 — 2자 이상이므로)

        return candidates
```

### 3.3 점수 체계 변경

| 패턴 유형 | 매칭 강도 | 점수 |
|----------|----------|------|
| 외국 인명 + 종목명 EXACT | 최강 | 0.55 |
| 외국 인명 + first 2 syllables | 강 | 0.55 |
| 외국 인명 + first 1 syllable | 강 | 0.40 (← 기존 1자 페널티 제거 → 0.21 → 0.40) |
| 일반 토큰 + EXACT | 강 | 0.55 |
| 일반 토큰 + first 2 | 중 | 0.40 |
| 일반 토큰 + first 1 | 약 | 0.30 |
| 1자 단일 토큰 (인명 패턴 외) | 노이즈 | **제외** (인명 패턴에서만 1자 허용) |

```python
def test_foreign_name_pattern_samtable():
    """샘 알트먼 → 샘표 매칭 (방법론 §2.4 핵심 사례)."""
    db = KRXStockDB.load_default()
    matcher = PhonologicalMatcher(db, min_marcap_eok=100, max_marcap_eok=5000)
    event = TriggerEvent(
        event_id="test", source=EventSource.RSS,
        title="샘 알트먼 OpenAI CEO 방한 AI 정상회담",
        body="", url="",
        published_at=datetime.now(timezone.utc),
        collected_at=datetime.now(timezone.utc),
        category=EventCategory.PERSON,
        keywords=["CEO", "방한", "AI"],
    )
    results = matcher.match(event)
    samtable = [c for c in results if c.stock_code == "007540"]
    assert len(samtable) > 0, "샘표 매칭 실패"
    assert samtable[0].matched_on == "샘 알트먼", f"matched_on should be '샘 알트먼', got '{samtable[0].matched_on}'"
    assert samtable[0].score >= 0.40, f"외국 인명 점수 >= 0.40, got {samtable[0].score}"


def test_foreign_name_pattern_others():
    """팀 쿡 → 쿡(없음, 음운만), 일론 머스크 → 일x, 트럼프 → 트x."""
    db = KRXStockDB.load_default()
    matcher = PhonologicalMatcher(db, min_marcap_eok=100, max_marcap_eok=5000)
    event = TriggerEvent(
        event_id="test", source=EventSource.RSS,
        title="일론 머스크 테슬라 CEO, 한국 진출 발표",
        body="", url="",
        published_at=datetime.now(timezone.utc),
        collected_at=datetime.now(timezone.utc),
        category=EventCategory.PERSON,
        keywords=["CEO"],
    )
    results = matcher.match(event)
    # 일/론으로 시작 종목 → 일진머티리얼즈, 일진다이아 등
    iljin = [c for c in results if c.stock_name.startswith("일")]
    assert len(iljin) > 0, "'일' 시작 종목 매칭 누락"
    assert any(c.matched_on == "일론 머스크" for c in iljin), "matched_on에 인명 전체 누락"


def test_foreign_name_excluded_from_general_tokens():
    """인명 패턴 매칭 후 일반 토큰에서 같은 첫 음절은 중복 차단."""
    db = KRXStockDB.load_default()
    matcher = PhonologicalMatcher(db, min_marcap_eok=100, max_marcap_eok=5000)
    event = TriggerEvent(
        event_id="test", source=EventSource.RSS,
        title="샘 알트먼 방한 샘소나이트 신제품 출시",  # "샘" 시작 단어 2개
        body="", url="",
        published_at=datetime.now(timezone.utc),
        collected_at=datetime.now(timezone.utc),
        category=EventCategory.UNKNOWN,
        keywords=[],
    )
    results = matcher.match(event)
    # "샘 알트먼"이 우선 매칭, 일반 토큰에서 "샘소나이트"는 중복 차단되어야
    sam_matches = [c for c in results if c.matched_on.startswith("샘")]
    foreign = [c for c in sam_matches if "알트먼" in c.matched_on]
    general = [c for c in sam_matches if c.matched_on == "샘소나이트"]
    assert len(foreign) > 0, "외국 인명 우선 매칭 실패"
    assert len(general) == 0, "일반 토큰 중복 매칭이 차단되어야"
```
