## KNOW-EMBED-01. 임베드 차단 정책에 대처하는 fallback 전략

**문제**: 한국 금융 사이트(네이버·다음·팍스넷) 모두 X-Frame-Options 헤더로 iframe 차단. iframe 임베드 시도 시 빈 화면.

**대응 (검증된 패턴)**:
- 차트 = **TradingView 위젯** (X-Frame-Options 허용) → iframe OK
- 뉴스/종목토론 = **외부 링크 카드** → 새 탭 (`target="_blank"`)

**노하우**:
- "iframe 임베드 가능 vs 불가" 판정: `curl -sI <url> | grep X-Frame-Options`
- 차단 사이트는 새 탭이 표준 (강제 우회 시 보안 문제)
- 사용자 안내문 명시: "iframe 임베드를 차단하여 새 탭으로 엽니다"

## KNOW-TRADINGVIEW-01. KRX 종목의 TradingView 위젯 URL

**검증된 패턴**:
```
https://s.tradingview.com/widgetembed/
  ?symbol=KRX:{6자리_코드}
  &interval=D (D=일봉, 60=60분, W=주봉, M=월봉)
  &theme=dark
  &locale=kr
```

**노하우**:
- KOSPI/KOSDAQ/KOSDAQ GLOBAL/KONEX 모두 `KRX:` prefix
- 일부 코스닥 종목은 차트 데이터 부재 (위젯은 로드되지만 빈 화면)
- 사용자에게 fallback 안내: "차트 데이터 없으면 네이버 금융 링크 활용"

## KNOW-IFRAME-RESOURCE-01. iframe 닫을 때 리소스 정리

**상황**: 분석 패널 모달 닫을 때 iframe이 그대로 DOM에 남아있으면 백그라운드 리소스(JS 실행, network 요청) 지속.

**해결**:
```javascript
function closeAnalysisOverlay() {
  $('#analysis-overlay').classList.remove('visible');
  $('#analysis-body').innerHTML = '';  // iframe 제거
}
```

**노하우**:
- iframe은 단순 `display:none`만으로 리소스 정지 안 됨
- 다시 열 때 처음부터 로드 (TradingView는 빠르므로 부담 X)
- 향후 캐시 필요 시 별도 hide/show 패턴 검토

## KNOW-EXTERNAL-LINK-PATTERN-01. 링크 카드 다중 노출

**원칙**: 같은 정보를 여러 소스에서 제공 — 사용자가 선택.

**구현**:
- 뉴스 4개 (네이버 뉴스 + 메인, 다음, Google)
- 종목토론 3개 (네이버, 다음, 팍스넷)

**노하우**:
- 한국 단타 사용자는 네이버 종목토론을 가장 많이 봄 (default 첫번째)
- 팍스넷은 보조 (커뮤니티 다양성)
- Google 검색은 일반 뉴스용 (금융 외 보도)

## TRAP-ESC-MULTIPLE-LISTENERS. 여러 Esc 리스너 중복 호출

**문제**: 기존 `keydown` 리스너가 modalOverlay(help)와 event-overlay 닫음. 새로 추가한 리스너가 analysis-overlay만 닫음. 두 리스너 동시 작동.

**현재 상태**: 문제 없음 — Esc 누르면 세 모달 모두 닫힘 (의도된 동작).

**노하우**:
- document-level 키 리스너 여러 개 등록 OK (각자 자기 영역만 처리)
- 단 stopPropagation 불필요 (Esc는 글로벌 키)
- 향후 모달이 더 늘면 글로벌 ModalManager로 통합 검토

## KNOW-AUTO-MODE-DECISION-01. Auto mode에서의 결정 보고서 작성

**상황**: 사용자가 명시 결정을 안 한 큰 변경 (분석 패널). Auto mode 활성.

**대응**:
- 결정 보고서 작성 (옵션 비교 + 권장안 + 자율 진행 명시)
- AskUserQuestion 생략 (Auto mode 권장)
- 권장안으로 진행 + 사용자 피드백 시 조정

**노하우**:
- Auto mode = "reasonable assumption" 권한 부여 단계
- 큰 결정도 결정 보고서는 작성해야 (사후 추적성)
- 결정 보고서 마지막에 "자율 진행" 명시하여 의도 투명화
