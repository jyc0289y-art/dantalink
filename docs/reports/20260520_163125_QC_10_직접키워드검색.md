## 4. 시행착오 / 노하우

### KNOW-VIRTUAL-EVENT-01. 가상 트리거 이벤트 패턴
**문제**: matcher_engine은 events 배열을 입력으로 받음. 키워드만으로 매칭하려면?

**해결**: 키워드를 가상의 단일 트리거 이벤트로 변환
- title = 키워드 합친 문자열 (Layer A 음운 토큰 추출용)
- keywords = 사용자 입력 (Layer B 의미 매칭용)
- category = classify(title) 자동 추론

**노하우**:
- 기존 매칭 엔진 100% 재사용 (재구현 없음)
- 카테고리 수동 선택보다 자동 추론이 사용자 부담 감소
- 가상 이벤트 source = 'manual' 표시로 디버깅 용이

### KNOW-UPLOAD-DEEMPHASIZE-01. 안 쓰는 옵션의 처리
**상황**: UPLOAD는 사용자가 거의 안 쓸 것 — 제거 vs deemphasize

**선택**: 시나리오 토글에서 제거하되, 사이드바 file-zone은 유지
- 화면 첫 인상: KEYWORD SEARCH가 자연스러운 첫 옵션
- 고급 사용자(JSON 가진 사람): 사이드바 drag-drop 가능

**노하우**:
- 안 쓰는 기능도 완전 삭제보다 deemphasize가 안전
- 메인 UX 흐름에서 빼고 고급 영역으로 이동
- 향후 사용 패턴 데이터 보고 완전 제거 결정

### TRAP-SCENARIO-EXCLUSIVITY-01. 시나리오 간 상호 배타성
**문제**: 사용자가 KEYWORD SEARCH 선택 후 DEMO 클릭하면? KS section 숨김 + DEMO 데이터 로드.

**해결**:
```javascript
if (source === '__keyword_search__') {
  ksSection.classList.add('visible');
  return;  // 데이터 로드 X
} else {
  ksSection.classList.remove('visible');
}
```

**노하우**:
- 시나리오 전환 = 항상 KS section 숨김
- KEYWORD SEARCH 활성 시는 데이터 로드 안 함 (사용자 입력 대기)
- 매칭은 ▶ MATCH STOCKS 버튼으로 명시적 트리거

## 5. 사용 흐름

```
1. Stock Match 페이지 접속
2. 상단 🔍 KEYWORD SEARCH 클릭
3. 사이드바 상단에 "▸ Direct Keyword Search" 섹션 표시
4. 키워드 입력 (예: "백신") + Enter
5. (선택) 추가 키워드 (예: "에볼라")
6. ▶ MATCH STOCKS 클릭
7. 카드 그리드에 종목 후보 표시 + 우측에 매칭 근거
8. 카드 클릭 → 차트·뉴스·종목토론 분석 패널 사용 가능
```
