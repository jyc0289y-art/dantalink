## 4. 시행착오 / 노하우

### KNOW-DEFAULT-01. 기본값이 곧 가이드라인
**상황**: 사용자가 "키워드 없이도 진행하고 싶다"라고 명시.
- 이전: `keywords: ['AI', '에볼라', 'OpenAI', 'Samsung']` 디폴트
- 변경: `keywords: []` + `dataset: '__all__'`

**노하우**:
- 기본값은 시스템이 "이렇게 쓰세요"라고 권장하는 시그널
- 사용자 의도가 "watch list" → "discover" 방향으로 이동했다면 default도 따라가야 함
- 사용자가 매번 4개 키워드를 지우는 번거로움 제거 + Discover 모드 즉시 진입 가능

### KNOW-DEDUP-01. dedup은 데이터셋 합산이 1개여도 안전하게
**구현**:
```javascript
if (datasetsToFetch.length > 1) {
  // dedup
}
```
- 단일 데이터셋 fetch는 dedup 불필요 (이미 source에서 unique)
- ALL 모드만 dedup 적용 → 불필요한 연산 회피

### TRAP-FETCH-FAIL-01. 다중 fetch에서 일부 실패 시 처리
**상황**: ALL 모드에서 events_rss.json 성공 + events_demo.json 실패
- 기존 코드는 모두 실패 시에만 abort
- 일부 성공 시: `totalLoaded > 0` 조건으로 진행 (best effort)
- 각 데이터셋 결과 로그 명시 → 사용자가 일부 실패 인지 가능
