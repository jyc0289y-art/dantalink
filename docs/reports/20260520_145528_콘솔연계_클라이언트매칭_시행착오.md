## 2. 신규 발견 / 노하우

### TRAP-CLIENT-MATCH-01. CORS 차단으로 외부 RSS 직접 fetch 불가
**원인**: 브라우저 same-origin policy. 외부 도메인(yna.co.kr, mk.co.kr 등)의 RSS는 CORS 헤더 없음.

**해결**:
- Data Collection Console은 "Pre-collected Dataset" 선택 방식으로 전환
- events_rss.json (Python CLI로 실제 수집) + events_demo.json (방법론 시연용)을 같은 origin에 두고 fetch
- UI는 "라이브" 느낌으로 키워드/소스 필터링 동작
- 진짜 라이브 수집은 Python `dantalink scan` CLI에서만 수행 (CORS 제약 없음)

**노하우**:
- 정적 사이트 GUI는 라이브 백엔드 없이도 "라이브처럼" 보일 수 있음 — pre-baked dataset + 클라이언트 필터
- 진짜 라이브는 별도 백엔드 도입 시 (FastAPI + WebSocket)

### KNOW-BRIDGE-01. localStorage 브리지 패턴
**상황**: 두 HTML 페이지(SPA가 아닌 별도 페이지) 간 데이터 전달.

**대안 비교**:
| 방법 | 적합도 | 비고 |
|------|--------|------|
| URL params | ✗ | 큰 데이터 (events 20건+) 불가, URL 길이 제한 |
| postMessage | △ | 새 창/iframe만 작동, 일반 navigation에는 X |
| **localStorage** | ✅ | 같은 origin이면 페이지 navigation 후에도 유지 |
| IndexedDB | △ | 과잉. localStorage 충분 (현재 페이로드 ~30KB) |
| sessionStorage | △ | 새 탭 열기 시 사라짐. 사용자 흐름과 충돌 |

**구현 패턴**:
```javascript
// 보내는 쪽 (Collector)
localStorage.setItem('dantalink_events', JSON.stringify(payload));
window.location.href = 'stock_match_gui.html?source=collector';

// 받는 쪽 (Matcher)
const params = new URLSearchParams(window.location.search);
if (params.get('source') === 'collector') {
  const payload = JSON.parse(localStorage.getItem('dantalink_events'));
  // 자동 매칭 실행
}
```

**노하우**:
- URL query param은 "어디서 왔는지" 신호용으로 사용 (`?source=collector`)
- 실제 데이터는 localStorage에 저장 (별도 key)
- 페이지 로드 시 자동 매칭 트리거 → 사용자 클릭 없이 즉시 결과 표시

### KNOW-PORT-01. Python 매칭 로직 JS 포팅
**원칙**: 점수·알고리즘 100% 동일 유지 (서버사이드와 클라이언트사이드 결과 같아야 함).

**포팅 매핑**:
| Python | JavaScript |
|--------|-----------|
| `@dataclass` | `class` + plain object |
| `defaultdict(list)` | `Map` + `get/set` boilerplate |
| `re.findall(r"[가-힣]{1,5}", text)` | `text.matchAll(/[가-힣]{1,5}/g)` |
| `re.match(r"^[\x00-\x7F]+$", kw)` | `/^[\x00-\x7F]+$/.test(kw)` |
| `set` | `Set` |
| `sorted(..., key=lambda)` | `.sort((a,b) => ...)` |

**점수 상수 (Python ↔ JS)**:
- 0.55 EXACT_MATCH ✓
- 0.40 FIRST_TWO ✓
- 0.30 FIRST_ONE ✓
- 0.70 SINGLE_TOKEN_PENALTY ✓
- 0.50 KEYWORD_SUBSTRING ✓
- 0.40 THEME_ONLY ✓

**노하우**:
- JS Map과 Python defaultdict 동작 차이 주의: JS는 `get` 후 `null` 체크 + `set` 필요
- 정규식은 거의 동일하지만 word boundary `\b`는 ASCII만 작동 (한글 word boundary 별도 처리 X)
- KRX DB 2,878건 + bigram 인덱스 → 메모리 ~5MB, 매칭 실행 100ms 이내

### KNOW-DATA-FORMAT-01. CSV → JSON 압축 (array form)
**관찰**:
- CSV (data/krx_stocks.csv): 130KB
- JSON object form (`[{c,n,m,k}]`): 230KB
- JSON array form (`[[c,n,m,k]]`): 116KB
- ↑ array form이 object form 대비 50% 작음 (key 반복 없음)

**구현**:
```python
# data/krx_stocks.json 생성
json.dump(
  [[row['Code'], row['Name'], row['Market'], int(row['Marcap_억원'])] for row in csv],
  f, ensure_ascii=False, separators=(',', ':')
)
```

```javascript
// JS에서 객체 형태로 복원
this.stocks = data.map(([c, n, m, k]) => ({ code: c, name: n, market: m, marcapEok: k }));
```

**노하우**:
- 컴팩트 JSON은 첫 로드 시간 최적화 (Pages 빌드 시 gzip 적용 → 실 transfer ~30KB)
- 별도 압축 없이 array form만으로 충분
- 단점: 컬럼 의미가 코드에만 명시됨 → README/주석 명확히
