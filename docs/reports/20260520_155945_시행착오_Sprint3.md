## KNOW-DISCOVER-01. Watch List → Discover로의 패러다임 전환

**상황**: 처음 시스템은 "키워드 입력 → 이벤트 필터" 흐름 (Watch List). 사용자 지적으로 "사용자가 모르는 키워드를 자동 발굴" (Discover) 흐름이 진짜 단타 알파에 부합함을 인지.

**구현 결과**:
- 두 흐름 모두 지원
  - Watch (사용자가 키워드 알 때): 입력 → 필터 → Forward
  - Discover (모를 때): 빈 상태 → SCAN → 트렌딩 chip → 관심 chip 선택 → Add to Watch → 좁힌 SCAN → Forward
- 사용자가 어떤 의도로 와도 매끄럽게 전환

**노하우**:
- 새 패러다임이 기존 흐름을 대체하는 게 아니라 "보강"으로 추가
- 두 흐름 사이의 다리 = Add Selected to Watch (Discover → Watch 자연스러운 promote)
- 사용자 경험상 "발견 → 좁히기" 패턴이 직관적

## KNOW-FREQUENCY-CATEGORY-01. 키워드별 카테고리 매핑

**문제**: 트렌딩 키워드만 보면 부족 → "어느 카테고리인지" 정보가 단타 결정의 핵심.

**해결**:
- `catCountPerKw` Map — keyword당 카테고리별 빈도
- 가장 흔한 카테고리를 그 키워드의 대표 카테고리로
- chip 좌측 보더 색상으로 시각화 (tcat-health/tech/geopolitical/person/disaster/regulatory)

**예시**:
- AI (21회) → tech (대부분이 tech 이벤트)
- 이란 (3회) → geopolitical
- 백신 (2회) → health

**노하우**:
- 빈도 단순 표시보다 "어디서 온 트렌드인가" 정보가 의사결정 정확도 증대
- 같은 키워드가 여러 카테고리에 등장하면 가장 흔한 것 채택 (다수결)

## KNOW-PROMOTE-PATTERN-01. chip 선택 → Watch promote 패턴

**구현**:
- chip click → Set에 추가 (`STATE.trendingSelected`)
- "+ Add Selected to Watch" 버튼 클릭 → 선택 chip들을 `STATE.keywords`로 이동
- 자동으로 trendingSelected 비움 + renderChips/renderTrending 재호출

**노하우**:
- multi-select 패턴은 Set 자료구조가 최적 (중복/추가/삭제 O(1))
- promote는 명시적 버튼 (실수 방지 — chip 클릭만으로는 안 되게)
- promote 후 시각 피드백 (Watch chip 자동 갱신 + 콘솔 로그)

## TRAP-EMPTY-SECTION-01. 빈 상태 처리

**상황**: 초기 페이지 로드 시 collectedEvents=[] → trending도 빈 배열.

**문제**: 빈 chip 영역 + 액션 버튼이 노출되면 UX 어색.

**해결**:
- `.trending-section.empty { display: none; }` CSS 적용
- `renderTrending()`에서 trending 길이 0이면 `add('empty')`, 있으면 `remove('empty')`

**노하우**:
- 빈 상태 UI 처리는 "숨김" + "안내 메시지" 두 패턴
- 첫 방문자 = 빈 상태 → 가급적 숨김 (UI 부담 감소)
- 사용자가 SCAN 후 비었다면 → "no matches" 안내 (현재는 events에서 처리)

## KNOW-ITERATION-01. Discover → Filter 반복 흐름

**워크플로우**:
```
빈 SCAN (Discover)
  → 트렌딩 12개 chip
  → 사용자가 3개 선택 (예: 이란, 백신, 자율주행)
  → Add to Watch
  → 다시 SCAN (Filter)
  → 더 좁은 결과 (예: 8건)
  → 트렌딩 12개 (좁아진 풀에서)
  → 사용자가 추가 선택 가능
  → ...
```

**노하우**:
- 1회 스캔으로 완벽한 키워드 발견은 어려움
- 반복 narrow-down 패턴 = 사용자 직관 + 시스템 정확도 결합
- 매 반복마다 콘솔 로그로 좁아짐 명시 (사용자 인지)

## KNOW-CSS-GRADIENT-01. 섹션 강조 디자인

**상황**: Discover 섹션이 다른 영역과 구분되어야 함.

**적용**:
```css
.trending-section {
  background: linear-gradient(135deg,
    rgba(78,205,196,0.06) 0%,
    rgba(255,182,39,0.04) 100%);
  border-left: 2px solid var(--cyan);
}
```

**노하우**:
- 그라데이션은 미묘하게 (alpha 0.04~0.06) — 너무 강하면 시각 노이즈
- 단색보다 그라데이션이 "활성 영역" 느낌
- 보더 색상 = 섹션의 기능 (Discover = 시안 = HUD 정보)
