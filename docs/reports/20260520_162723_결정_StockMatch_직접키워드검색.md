## 2. 키워드 검색 구현 옵션

### 알고리즘
키워드 N개 → 가상의 단일 트리거 이벤트로 변환:
```
event = {
  event_id: 'keyword_search',
  source: 'manual',
  title: 키워드 합친 문자열,
  body: '',
  category: classify(키워드 합산).category,  // 자동 추론
  keywords: 사용자 입력 키워드[],
  ...
}
```
→ matcher_engine.match([event])로 실행 → top_stocks 결과 표시

이 방식의 장점:
- 기존 매칭 엔진 100% 재사용 (포팅된 Python 로직)
- Layer A 음운 (title에서 토큰 추출) + Layer B 의미 (keywords에서 substring)
- 카드 인사이트(EXACT/Dual/GENUINE/NAME-ONLY) 자동 적용

### UI 옵션

| 옵션 | 형태 |
|------|------|
| A | 시나리오 토글에 KEYWORD SEARCH 추가, 클릭 시 사이드바 상단에 검색 섹션 표시 |
| B | 사이드바 최상단에 항상 키워드 검색 섹션 (다른 시나리오와 무관하게) |
| C | 헤더에 검색 input (Google처럼) |

**채택: A** — 다른 시나리오와 동일한 UX 패턴 (토글 → 콘텐츠), 사용 안 할 때는 영역 차지 X
