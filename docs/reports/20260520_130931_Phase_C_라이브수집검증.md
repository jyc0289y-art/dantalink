## 2. GDELT 라이브 수집 — ⚠️ 윈도우 IP 일시 차단

### 시도 결과
- 첫 호출 (4 키워드): 모두 429 Too Many Requests
- 1분 대기 후 단일 키워드 재시도: 429 지속
- 직접 httpx GET 호출 (User-Agent 명시): 429 지속

### GDELT 서버 명시 메시지
```
Please limit requests to one every 5 seconds or
contact kalev.leetaru5@gmail.com for larger queries.
```

### 코드 fix 적용 완료 (`dantalink/collectors/gdelt.py`)
1. `asyncio.sleep(5.5)` 호출 간 대기 (5초 한도 + 안전 마진)
2. `User-Agent: DantaLink/0.1 (+research)` 헤더 추가
3. 위 두 가지를 `GDELT_INTER_QUERY_SLEEP_SECONDS` 상수로 명시

### 후속 조치
- **현 윈도우 세션**: 차단이 ~10분 이상 지속될 가능성. RSS만으로 Phase C 검증 통과 처리.
- **맥북 세션**: 다른 IP에서 즉시 재검증 가능. 5.5초 sleep으로 안전.
- **장기적 (M2)**: exponential backoff + per-IP burst counter (plan H3/M2 통합)

## 3. cli.py 인코딩 fix (윈도우 cp949 콘솔 호환)

### 문제
```
UnicodeEncodeError: 'cp949' codec can't encode character '✓' (✓)
```
cli.py가 `✓`(✓), `▶`(▶), `\U0001f4be`(💾) 등을 출력하지만 윈도우 콘솔 기본 인코딩(cp949)이 수용 못 함.

### 적용 fix (`dantalink/cli.py:127-133`)
```python
def main() -> int:
    # 윈도우 콘솔(cp949)에서도 ✓/▶/💾 같은 유니코드 출력이 깨지지 않도록 UTF-8 강제
    if sys.stdout.encoding and sys.stdout.encoding.lower() != "utf-8":
        try:
            sys.stdout.reconfigure(encoding="utf-8", errors="replace")
            sys.stderr.reconfigure(encoding="utf-8", errors="replace")
        except (AttributeError, ValueError):
            pass
    ...
```

`PYTHONIOENCODING=utf-8` 환경변수가 우선이지만, 미설정 환경에서도 동작.
