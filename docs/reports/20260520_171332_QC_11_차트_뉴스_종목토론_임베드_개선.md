## 5. 시행착오 / 노하우

### TRAP-XFO-ASSUMPTION-01. X-Frame-Options 추측의 위험
**문제**: Phase B 작업 시 "한국 금융 사이트는 iframe 차단 정책"이라 추측 → 새 탭 링크로만 구현. 사용자가 "오버레이 내 브라우징 원함" 피드백 시 사전 검증 안 했던 가정이 잘못이었음을 발견.

**노하우**:
- `curl -sI <url> | grep X-Frame` 으로 **반드시 사전 검증**
- 도메인 단위 가정 X — 페이지별 헤더 다를 수 있음
- 모바일 페이지(`m.`) vs 데스크톱(`www.`) X-Frame 정책 다를 수 있음
- 추측이 아닌 확인 후 결정 보고서 작성

### KNOW-CHART-FALLBACK-01. 인터랙티브 + 정적 이중 fallback
**상황**: TradingView가 일부 KRX 종목 미지원 → 사용자 좌절.

**해결**: 3소스 토글
- 네이버 모바일 (인터랙티브, 모든 한국 종목)
- TradingView (인터랙티브, 일부 종목)
- 정적 이미지 (모든 종목 보장, 무겁지 않음)

**노하우**:
- "한 가지에 다 걸지 않음" — 핵심 기능은 fallback 체인 구성
- default는 가장 광범위 작동하는 소스 (네이버 모바일)
- 인터랙티브 실패해도 정적 이미지로 최소 동작 보장

### KNOW-EMBED-LAYOUT-01. 메인 iframe + 사이드 대체 링크
**구조**:
```
┌────────────────────┬──────────┐
│ Primary iframe     │ Alt 200px│
│  (네이버 모바일)   │ 링크 카드│
└────────────────────┴──────────┘
```

**노하우**:
- iframe이 빈 화면이거나 깨져도 사이드의 대체 출처로 즉시 전환 가능
- 새 탭으로 떠나는 게 아니라 옆에 항상 표시 — 사용자 안전망
- 200px 정도면 너무 좁지도 넓지도 않음

### KNOW-URL-REDIRECT-01. 301 추적은 fetch 전에
**관찰**: `/discuss` → 301 → `/discussion`. fetch 시 자동 follow되지만 iframe 임베드 시 첫 응답 사용 → 빈 화면 가능.

**해결**: 최종 URL(`discussion`)을 직접 사용.

**노하우**:
- iframe src는 301 follow 안정성 낮음 → 최종 URL 사용
- `curl -sIL` 로 redirect 체인 추적 후 최종 path 채택
