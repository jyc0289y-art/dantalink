## 📝 진영님께 검토 요청 사항

### 1️⃣ 시나리오 A (방법론 사례)
- [ ] 아이진 1위 매칭이 본인이 인지하던 "백신 카테고리 매핑" 사례와 일치하는가?
- [ ] 해성에어로보틱스가 호르무즈 → 방산 시그널로 합리적인가?
- [ ] 다른 누락된 종목이 있는가? (예: 아이진 외 다른 백신주, 다른 방산주)

### 2️⃣ 시나리오 B (라이브 RSS)
- [ ] 비상교육·코아스가 실제 단타 후보로 활용 가능한가?
- [ ] 반도체·로보틱스 종목 다수 노출은 시그널인가 노이즈인가?
- [ ] 단타 빈도와 견조한 거래량 종목 위주로 필터링이 필요한가? (Stage 03 진입 신호 검증 영역)

### 3️⃣ 정책 결정 필요
- [ ] **샘표 케이스**: 1자 토큰 페널티 (현재 0.7) 완화 또는 외국 인명 패턴(`샘 알트먼` 형태) 별도 추출?
- [ ] **시총 정책**: 현재 max-marcap 5,000억. 대형주(한화에어로·LG에너지솔루션 등) 함께 보고 싶으면 `--max-marcap 0`
- [ ] **GDELT**: 윈도우 IP 차단 상태. 맥북 IP에서 재검증하려면? (코드는 5.5초 sleep 적용 완료)

### 4️⃣ 사용자가 직접 시도 가능한 명령
```powershell
cd C:\Users\jyc\iCloudDrive\iCloudDrive\developer\DantaLink

# 라이브 RSS 다시 수집 (시점 변경)
$env:PYTHONIOENCODING="utf-8"
python -m dantalink.cli scan --keywords "AI,Samsung,에볼라,OpenAI" --timespan 24h --sources rss --max-records 30 --output events_rss.json

# 매칭 (현재 default: max-marcap 5000)
python -m dantalink.cli match --input events_rss.json --output matches.json --top 30 --min-marcap 100 --max-marcap 5000

# 대형주 포함하여 시각화
python -m dantalink.cli match --input events_rss.json --output matches_all.json --top 50 --min-marcap 0 --max-marcap 0 --sort total

# 시연 시나리오 (방법론 §2.4)
python -m dantalink.cli match --input events_demo.json --output matches_demo.json --top 30 --min-marcap 100 --max-marcap 5000
```
