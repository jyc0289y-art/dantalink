## 🔍 사용자 의견 타당성 우선 검토

### Issue 2: 콘솔 연결 방향 — **사용자 의견이 타당함** (자기 비판 인정)

#### 현재 흐름 (제가 만든 것)
```
[Data Collection Console]
  사용자가 키워드 수동 입력 ("AI, 에볼라, OpenAI, Samsung")
       ↓
  사전 수집된 events_rss.json fetch + 키워드 필터링
       ↓
  필터된 이벤트들을 Stock Match로 forward
       ↓
[Stock Match Console]
  이벤트의 카테고리·키워드로 매칭
```

이건 **"Watch List 방식"** — 사용자가 이미 관심 키워드를 알고 그것을 모니터링하는 구조.

#### 사용자가 원한 흐름 (방법론.md §1.1과 일치)
```
[Data Collection Console]
  RSS·GDELT 등 트리거 소스 스캔
       ↓
  핫 키워드 / 트렌딩 이벤트 자동 발굴 ← 여기가 핵심
       ↓
  발굴된 키워드(예: "분디부교", "샘 알트먼")를 Stock Match로 forward
       ↓
[Stock Match Console]
  그 키워드로 종목 매칭
```

이건 **"Trending Discovery 방식"** — 모르는 트리거를 자동으로 발견하는 구조.

#### 왜 제가 반대로 만들었는가 (자기 비판)
1. Python CLI `dantalink scan --keywords "AI,에볼라"` 가 키워드 입력 방식 (방법론 §1.1보다 단순한 형태)
2. 기존 `DantaLink_데이터수집기.html` 프로토타입도 키워드 입력 + 토글 디자인
3. 그 기존 패턴을 그대로 옮겨 옴 → **방법론의 진짜 의도(트리거 발굴)를 놓침**
4. 사용자가 처음 방법론.md §1.1에 "**트리거 이벤트 감지**"라 적었음에도 구현에서 "**키워드 입력**"으로 단순화

#### 단타 알파 관점에서의 비교

| 측면 | Watch List 방식 (현재) | Trending Discovery (사용자 의도) |
|------|---------------------|----------------------------|
| 사전 예측 부담 | 사용자가 키워드를 미리 알아야 | 자동 발굴 |
| 첫 등장 이벤트 (예: 새 인물 방한) | ❌ 키워드 없으면 놓침 | ✅ 자동 포착 |
| 알림·모니터링 (이미 관심 종목) | ✅ 적합 | △ 노이즈 가능 |
| 방법론.md §1.1 부합 | △ 부분 부합 | ✅ 완전 부합 |
| 진영님 사례 (분디부교 PHEIC) | △ "분디부교" 키워드 미리 알았어야 | ✅ "WHO/PHEIC" 키워드만 있어도 발견 |

→ **결론**: 단타 발굴의 진짜 가치는 Trending Discovery. Watch List는 보조.

## 🎨 Issue 1: Stock Match GUI 인사이트 강화

### 사용자가 못 본 핵심 정보 5가지

클로드 챗 평가와 현재 GUI 비교:

| 평가에 명시된 정보 | 현재 GUI 표시 여부 | 누락 원인 |
|---------------|----------------|---------|
| **Dual Match (음운+의미 동시)** 강조 | △ 레이어 배지 2개 표시되지만 강조 없음 | 단순 나열 |
| **신뢰도 등급** (0.55=★★★★, 0.40=★★) | ❌ 숫자만, 등급 시각화 없음 | 별점/계급 UI 부재 |
| **사명 매칭의 부작용** (사피엔반도체 = 반도체 박스) | ❌ 표시 없음 | 기능 자체 미구현 |
| **각 종목의 발굴 메커니즘 한 줄 요약** | ❌ Match Evidence 클릭해야 보임 | 카드 자체에는 요약 없음 |
| **카테고리 박스 정확도 경고** | ❌ 없음 | 미구현 |

### 클로드 챗 평가가 한눈에 보여준 것

```
사피엔반도체  → Dual Match (음운+의미)  → 신뢰도 0.55 → ★★★★ 가장 안정
진원생명과학  → 카테고리만             → 신뢰도 0.45 → ★★★  자체 호재 보정
엔로보틱스    → 음운만                 → 신뢰도 0.30 → ★★   가장 휘발성
```

이 정보는 현재 카드에서도 max_score 숫자(0.55/0.50/0.40)와 layers 배지로 표현은 됨. 그러나 **"이게 어떤 의미인지"의 직관적 변환이 안 됨**.

## 📋 3가지 구현 옵션 (Issue 1 + Issue 2 통합)

| 옵션 | Issue 1 (GUI 인사이트) | Issue 2 (연결 방향) | 구현 비용 | 다음 단계 부담 |
|------|---------------------|------------------|---------|------------|
| **A** | 카드 정보 강화 (Dual 배지, 신뢰도 등급, 별점) | 현재 watch list 유지 + 사용자 의도 별도 작업 | ★★☆☆☆ (2h) | 작음 |
| **B** | A + 매칭 메커니즘 한 줄 요약 + 사명 매칭 경고 | Trending Discovery 모드 추가 (양방향 토글) | ★★★★☆ (5h) | 중간 (트렌딩 알고리즘) |
| **C** | B + 새 "INSIGHTS" 패널 (Top 3 자동 분석) | 사용자 의도(Trending)를 default로 + watch list는 옵션 | ★★★★★ (8h+) | 큼 (재설계) |

### 옵션 A 상세 — 빠른 GUI 개선만
- 종목 카드:
  - "Dual Match" 빨간 배지 (음운+의미 동시 매칭 시)
  - 신뢰도 ★★★★/★★★/★★ 별점 (max_score 기반)
  - 매칭 메커니즘 한 줄 요약 ("음운+카테고리 동시" 등)
- Issue 2 (연결 방향)은 **이 작업 후 별도 결정 보고서**로 처리

### 옵션 B 상세 — 양방향 지원
- 옵션 A의 GUI 개선 모두
- Data Collection Console에 **모드 토글** 추가:
  - 🔍 **Discover Mode** (default, 사용자 의도) — RSS에서 핫 키워드 자동 추출 + 카드로 표시 + 사용자 확인
  - 📋 **Watch Mode** (현재) — 사용자가 키워드 직접 입력
- Discover Mode 알고리즘 (단순):
  - RSS 데이터에서 분류기 keywords 추출
  - 빈도 ≥ N (예 3건 이상) 키워드 자동 후보
  - 카테고리별로 그룹
  - 사용자가 chip 클릭으로 선택 → Forward

### 옵션 C 상세 — 전면 재설계
- 옵션 B + 새 "INSIGHTS" 패널 (Stock Match 우측):
  - Top 3 종목 자동 분석 (클로드 챗 평가 스타일)
  - "이 종목은 사명 매칭 부작용 가능" 경고 자동 생성
  - 종목별 매수 가이드 (Kelly 사이징 자동 계산)
- Stock Match 메인 default를 "Discover에서 받은 키워드" 시나리오로

## 💡 권장안: **옵션 B**

### 권장 이유
1. **사용자 의도 직접 반영** — Trending Discovery 모드를 default로
2. **기존 Watch Mode도 보존** — 모니터링/알림 유스케이스 살림
3. **GUI 인사이트 강화는 즉시 효과** — Dual 배지·별점·한 줄 요약으로 평가급 정보 제공
4. **C(인사이트 패널)는 별도 분기** — 사이징 자동 추천 등은 더 큰 작업이므로 분리
5. **구현 부담 합리적** — 5시간 내 완료 가능

### 권장하지 않은 이유
- A: GUI 개선만으로는 사용자 핵심 요구(연결 방향)를 안 해결
- C: 매수 가이드·사이징 자동 계산은 자동매매 영역 → 별도 검토 필요 (CLAUDE.md 절대 금지 사항 §1 "자동 매수 코드 활성화 금지" 인접 영역, 신중 결정 필요)
