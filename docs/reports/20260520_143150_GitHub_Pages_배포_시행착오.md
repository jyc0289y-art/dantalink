## 2. 발견된 함정 (신규 TRAP 식별)

### TRAP-DEPLOY-01. iCloud 동기화 충돌로 파일명 " 2" suffix
**증상**:
- 본 세션에서 `Write docs/DantaLink_종목추출_GUI.html` 두 번 호출 (첫 작성 + 툴팁 추가 재작성)
- iCloud Drive가 두 번째 Write를 conflict로 판단 → `DantaLink_종목추출_GUI 2.html`로 새 파일 생성
- 원본 `DantaLink_종목추출_GUI.html`은 사라지고 " 2" 버전만 git에 staged

**진단**:
```bash
$ git ls-files docs/*.html
docs/DantaLink_데이터수집기.html
docs/DantaLink_시각화.html
docs/DantaLink_종목추출_GUI 2.html  # ← " 2" suffix 발견
```

**원인**:
- iCloud Drive는 같은 파일에 짧은 시간 간격으로 여러 write가 발생하면 conflict resolution으로 " N" suffix 추가
- 두 번째 Write 사이에 iCloud 동기화가 끼어듦
- Claude Code의 Write 도구는 파일을 in-place 덮어쓰지 않고 atomic rename 가능성

**해결**:
- `docs/DantaLink_종목추출_GUI 2.html`을 정식으로 인정 (이미 git에 포함)
- 영문 alias 추가: `docs/stock_match_gui.html` (cp via PowerShell `-LiteralPath`로 공백 escape)
- index.html link도 영문 alias로 변경

**노하우**:
- iCloud 경로에서 같은 파일 다회 Write 시 충분한 간격 또는 atomic operation 보장 필요
- 파일명에 공백·특수문자 들어가면 PowerShell `-LiteralPath` 또는 bash `"quoted"` 필수
- git에 한 번 들어간 파일명은 변경 어려움 → 영문 alias가 가장 안전한 회피
- 향후: Claude Code Write 도구는 iCloud 환경에서 짧은 대기 시간을 자체 적용하면 좋음

### TRAP-DEPLOY-02. GitHub Pages 한글 URL 404
**증상**:
- 푸시된 한글 파일명 `docs/DantaLink_종목추출_GUI 2.html` (95KB, 정상 존재)
- 브라우저 URL `docs/DantaLink_%EC%A2%85%EB%AA%A9%EC%B6%94%EC%B6%9C_GUI%202.html` (URL encoded)
- 응답: **HTTP 404 Not Found**

**원인 추정**:
- iCloud + macOS APFS는 한글을 NFD 정규화 (자모 분리: ㅈ+ㅗ+ㅇ+ㅁ+ㅗ+ㄱ)
- Windows NTFS + GitHub Pages는 NFC 정규화 (음절 단위: 종+목)
- URL encoding 시 어느 한쪽 정규화만 매칭 → 404
- (확정 진단 아님, 추정 — 빠른 해결을 위해 영문 alias로 우회)

**해결**:
- 영문 alias 파일을 추가로 push (`stock_match_gui.html`, `data_collector_gui.html`, `methodology_viz.html`)
- 한글 원본도 그대로 유지 (로컬 + iCloud 일관성)
- index.html 링크는 영문으로

**노하우**:
- GitHub Pages에 한글 파일명 푸시 금지 (또는 영문 alias 항상 동반)
- 사용자 글로벌 지침의 한글 파일명 정책은 로컬 작업 보고서에만 적용
- public-facing artifact (GUI HTML, JSON 산출물) 파일명은 영문이 안전
- 차후: 공개 산출물은 처음부터 영문, 내부 보고서·문서는 한글 — 컨벤션 분리

### TRAP-DEPLOY-03. Jekyll 기본 빌드의 한글 처리
**증상 (1차 시도)**:
- Pages 활성화 직후 root `/` 200이 아닌 404
- README.md를 Jekyll이 인덱스로 변환해야 하지만 그렇지 않음

**원인**:
- `.nojekyll` 파일 추가로 Jekyll 비활성화 → README.md 자동 인덱스 변환 X
- root에 `index.html`이 없으면 / → 404

**해결**:
- `index.html` 랜딩 페이지 작성 (Terran Console 디자인, 4개 GUI Quick Launch)
- push → 빌드 → / 200 OK

**노하우**:
- `.nojekyll` 사용 시 root에 `index.html` 필수
- `.nojekyll` 없이 Jekyll 사용 시 한글 파일명·특수문자 처리 불안정 → 영문 alias + `.nojekyll` 조합이 가장 안전
- Jekyll 비활성화 = 단순 정적 서빙 = 한글 path도 잘 처리되지만 NFC/NFD 정규화 이슈는 별개
