## KNOW-JOIN-01. 단순 title 기반 join이 의외로 견고

**상황**: 매칭 카드(`matches.json`의 matches[])는 event_title만 보유. 원본 본문/URL을 보려면 events_*.json과 join 필요.

**선택지**:
- event_id 기반 join — 더 robust지만 matches.json에 event_id가 없음 (matches[]에는 event_title만)
- title 기반 join — 동일 title이 여러 source에 존재하면 충돌

**결과**: title 기반으로 20건 모두 join 성공. 한국 RSS는 헤드라인 중복이 적어 충돌 0건.

**노하우**:
- Stage 02 매칭 결과를 향후 수정 시 event_id를 matches[]에 포함하면 더 안전
- 지금은 title join으로 충분 — 사용자 경험상 문제 X

## KNOW-OVERLAY-01. 닫기 3중 안전망

**상황**: 모달 사용자 경험은 "쉽게 열고 쉽게 닫기"가 핵심.

**구현**:
1. ✕ 버튼 클릭 (가장 명확)
2. 외부 overlay 클릭 (모달 외 어두운 영역)
3. Esc 키 (키보드 사용자)

**구현 디테일**:
- 외부 클릭 판별: `if (e.target === $('#event-overlay')) close()` — modal 자체 클릭은 무시 (event bubbling 활용)
- Esc 키는 document level — help 모달 + event 모달 둘 다 닫음 (각각 조건 체크)

**노하우**:
- 모달 패턴은 항상 3중 닫기 (✕/외부/Esc) 표준화
- 모바일 환경에서는 swipe-down 추가 검토 가능

## TRAP-EVENT-PROPAGATION-01. 카드 안의 select 클릭이 카드 클릭으로 전파

**상황**: match-card 클릭 시 모달 열림. 그러나 카드 안에 SORT select가 있으면 select 조작 시 모달이 열리는 부작용.

**해결**:
```javascript
card.addEventListener('click', (e) => {
  if (e.target.closest('.match-sort-row')) return;  // sort 영역 클릭은 제외
  ...
});
```

**노하우**:
- 중첩 클릭 핸들러는 항상 event delegation 시 `closest()`로 origin 확인
- React 같은 framework은 stopPropagation으로 처리, vanilla JS는 closest 패턴이 깔끔

## KNOW-PRELOAD-01. 페이지 로드 시 비동기 데이터 사전 fetch

**상황**: 오버레이는 매칭 카드 클릭 시점에 열림. 그 시점에 events 데이터를 fetch하면 지연 발생.

**해결**: 페이지 로드 직후 `loadEventsForOverlay()` 비동기 호출. 사용자가 모달 열 때쯤이면 이미 map 구축 완료.

```javascript
// 페이지 로드 시 비동기로 events join map 준비 (블로킹 X)
loadEventsForOverlay();  // await 없음
```

**노하우**:
- "사용자가 인터랙션할 가능성이 높은 데이터"는 idle 시간에 사전 fetch
- await 없이 fire-and-forget — UI 블로킹 회피
- 사용자가 매우 빨리 클릭하면 join 실패 가능 → fallback "원본 join 실패" 메시지로 graceful degradation

## KNOW-SORT-LOCALE-01. 한국어 정렬은 localeCompare(ko)

**상황**: title 정렬 시 한국어 단어 순서.

**구현**:
```javascript
.sort((a, b) => (a.title || '').localeCompare(b.title || '', 'ko'))
```

**노하우**:
- 단순 `<` 비교는 유니코드 codepoint 순서 → 한국어 가나다 순서와 불일치
- `localeCompare(b, 'ko')` 명시 시 ㄱ→ㅎ 순서 정상
- 영문은 default localeCompare도 ok이지만 ko 명시가 안전
