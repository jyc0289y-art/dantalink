## 2. matchers/ 모듈 구조

```
dantalink/matchers/
├── __init__.py         # 패키지 export
├── base.py             # BaseMatcher ABC + safe_match()
├── models.py           # MatchCandidate dataclass
├── korean_stocks.py    # KRXStock + KRXStockDB (음절·bigram 인덱스)
├── phonological.py     # Layer A — 음운 매칭
└── semantic.py         # Layer B — 의미 매칭
```

### 2.1 KRXStockDB 인덱스
- `by_first_syllable["샘"] → [샘씨엔에스, 샘표, 샘표식품, 빛샘전자]` — Layer A용
- `by_substring["백신"] → [중앙백신, 차백신연구소]` — Layer B용 (bigram 2자 기반)
- 330개 시작 음절, 4,461개 bigram

### 2.2 Layer A — Phonological (음운 매칭)
- 트리거 헤드라인을 regex `[가-힣]{1,5}`로 토큰화
- 1자 토큰 stopword + 2자+ stopword 두 종류 필터 (false positive 차단)
- 첫 음절 일치 → 종목 후보
- 점수: exact 0.55 > first2 0.40 > first1 0.30 (1자 토큰은 ×0.7 페널티)
- 시총 필터 (min/max) + max_per_token 한도
- 정렬: **점수 내림차순 + 시총 (동점 시)** — exact match가 시총 큰 종목보다 우선

### 2.3 Layer B — Semantic (의미 매칭)
- 이벤트 키워드(한국어 2자+) → 종목명 substring 직접 매칭 (0.50)
- 카테고리 → 테마 키워드 사전 → 종목명 substring (0.40)
- 카테고리 매핑: HEALTH=[바이오·제약·백신·메디·팜·셀·헬스·한미·녹십자], TECH=[반도체·소프트·테크·…], GEOPOLITICAL=[방산·항공우주·에어로·조선], …

### 2.4 CLI `match` 서브명령
```bash
dantalink match --input events.json --output matches.json \
    --top 30 --min-marcap 100 --max-marcap 5000 \
    --sort max  # or 'total'
```
- 입력: scan 결과 JSON
- 출력: 종목별 점수·매칭·근거 + 전체 후보 리스트
- 정렬: max=단일 최고 신호 우선 / total=총점
